# Интерфейс — приложение на устройстве

[English](interface.md) | **Русский**

Прошивка advui заменяет штатный экран Meshtastic клавиатурным UI, построенным под
Cardputer ADV. Это гид по тому, что на экране и как этим управлять.

Движок внизу — немодифицированный upstream Meshtastic, так что в меше нода ведёт себя как
обычное Meshtastic-устройство: меняется только локальный UI. В **режиме компаньона** тот же
UI управляет радио *другой* ноды по Bluetooth (см. в конце).

## Экраны

### Заставка

При загрузке — фирменный сплэш **`Meshtastic ADV`** на ~2 секунды (или до любой клавиши),
пока поднимается меш-движок. Дальше — Чаты.

### Чаты (дом)

Недавние переписки — личные и каналы вперемешку, свежие сверху.

```
 Chats                     [2 new]  78%    <- заголовок · непрочитанное · наша батарея
 ------------------------------------------
 ✉ SPb Gate                        14:02   <- конвертик · имя · время последнего
   hi, anyone near Nevsky?                 <- превью последнего сообщения
 # MediumFast                      13:47
   KSV: back online
   Neva Bridge                     12:30
   > see you there                         <- «>» = последнее сообщение — наше
 type find   Tab: all nodes
```

- **Строка = переписка.** Каналы с префиксом `#` (голубые), избранные — жёлтые, красный
  конвертик — непрочитанное. В превью каналов показан короткий ник отправителя, `>` — если
  последнее слово было за вами.
- **Enter** открывает переписку **на первом непрочитанном**.
- **← / →** — убрать из избранного / добавить выбранный контакт или канал.
- **Печать** ищет по *всем* узлам для нового чата; **Tab** — полный список узлов.
- В режиме компаньона маленькая **Bluetooth-руна** у заголовка показывает состояние линка
  (зелёная — подключено, жёлтая — переподключение, красная — обрыв).

### Список узлов (Tab)

Все, кого знает база узлов, по строке на узел.

```
 42 nodes                  [2 new]  78%    <- сколько узлов знает меш · батарея
 ------------------------------------------
 ✉ ksv-relay      ▂▄▆_   →1   4m    CLI
 * SPb Gate       ▂▄▆█   →0   now   RTR    <- * избранный (жёлтый)
   Neva Bridge    ▁___   →3   12m   RTR
 </>fav  ENTER open  type find  ESC/Tab chats
```

- **Сортировка:** непрочитанные → избранные → те, с кем переписка → остальные по хопам.
- **↑ / ↓** — курсор (список скроллится), **Enter** — открыть узел, **← / →** — избранное,
  **печать** — фильтр по имени, **ESC** или **Tab** — назад в Чаты.

Как читать строку:

| Поле        | Значение                                                                       |
| ----------- | ------------------------------------------------------------------------------ |
| **имя**     | длинное имя (иначе короткое, иначе `!nodenum`); жёлтое = избранный             |
| **сигнал**  | 0–4 полоски по последнему прямому SNR (зелёный сильный → оранжевый слабый); пустые = слышен только через ретрансляторы |
| **`→N`**    | хопов до узла: `→0` — прямой сосед · `→?` — неизвестно                          |
| **давность**| когда слышали: `now` · `5m` · `2h` · `3d`; зелёное — свежее, серое — старое      |
| **роль**    | `CLI` клиент · `RTR` роутер · `RPT` репитер · `TRK` трекер · `SEN` сенсор · `TAK` |

### Переписка

Экран треда — личка или канал. Шапка повторяет строку узла/канала, середина — сообщения,
внизу — строка набора.

```
 SPb Gate         ▂▄▆█   →0   now   RTR
 ------------------------------------------
 13:58 > Are you around?             ✓     <- наше сообщение · галка доставки
 [13:58 > Are you around?]                 <- цитата оригинала в рамке
 14:02 < yes! near the bridge              <- их ответ на него
 14:03 > 👍
 _                                    RU   <- строка набора · бейдж раскладки
 type  Tab emoji  <reply  >react
```

- **У каждого сообщения** — компактное локальное `HH:MM` (сдвиг UTC — в настройках) и
  маркер направления: `>` отправлено, `<` получено. В каналах у входящих виден короткий
  ник отправителя. Сообщения, пришедшие до синхронизации часов, запоминаются по моменту
  аптайма, и как только время появляется — задним числом получают настоящие метки (в
  пределах текущей загрузки; через потерю питания момент прилёта уже не восстановить —
  такие остаются без метки).
- **Статус доставки** у ваших сообщений: серая точка при отправке → **зелёная ✓** по
  routing-ACK (широковещательные получают ✓ по факту передачи) → **красный ✗ с причиной**
  при ошибке. Если последнее сообщение не ушло — **Enter отправляет его повторно**.
- **↑ / ↓** — прокрутка истории, **зажмите для непрерывной** (стрелки автоповторяются
  везде, backspace при стирании — тоже); открытие треда прыгает к первому непрочитанному.
- **Прокрутка выше верха уходит во флеш-архив**: вытесненные из живого окна сообщения не
  пропадают — новейшие 256 ложатся в архив во внутренней флеши, и ↑ на верху треда листает
  их страницами по 16 (в подвале `history N-M/K`; ↓ внизу страницы — к более новым, **ESC**
  возвращает в живой вид). Удаление чата стирает и его архив.
- **Печатайте** и жмите **Enter** для отправки. **Fn+L** переключает слой кириллической
  транслитерации (бейдж `RU`/`EN` на строке набора; выбор сохраняется). Раскладка
  фонетическая: `zh`→ж · `sh`→ш · `ch`→ч · `sch`→щ · `ya`→я · `yu`→ю · `yo`→ё · `ye`→э ·
  `y`→ы · `j`→й · `x`→ъ · `'`→ь · `h`→х · `c`→ц · `e`→е · `q`→я. **Tab** — палитра эмодзи
  (~24 иконки, стрелки + Enter для вставки; эмодзи рисуются прямо в тексте).
- **← / →** — режим *выбора сообщения*: ↑/↓ выбирают входящее, затем
  - **→ реакция** — быстрая лента тапбэков (`</>` — выбор, Enter — отправить). Чужие
    реакции (в том числе из телефонных приложений) видны под сообщением-адресатом.
  - **← ответ** — цитирует выбранное в вашей следующей отправке; в треде цитата рисуется
    в рамке над вашим сообщением — внутри время и отправитель оригинала (часто это его
    единственный след, когда оригинал уже вытеснен из истории), а телефонные приложения
    рисуют её как полноценный ответ.
  - На сообщения из совсем старой истории (сборки до реакций) реагировать нельзя — у них
    нет id пакета; подсказка внизу предупредит.
- **ESC** — назад. В правом углу подвала всегда — **наша батарея**.

### Настройки (долгое ESC откуда угодно)

Два уровня: верхнее меню показывает секции с живым превью ключевого значения; Enter
открывает секцию (или WiFi / MQTT / Radio напрямую), ESC поднимается обратно.

| Секция | Строки |
| ------ | ------ |
| **Node**   | **Name** и **Short** — длинное/короткое имя ноды (текстовый редактор)      |
| **LoRa**   | **Region** (обязателен при первом запуске) · **Preset** (LongFast, MediumFast, …) · **Frequency** (переопределение слота, МГц) · **Channel** (имя primary; ключ сохраняется) · **Role** (Client, Client Mute/Hidden, Router (Late), Repeater, Tracker, Sensor, TAK) · **Hops** (1–7) · **Power** (максимум региона или 2–22 dBm) · **Rebroadcast** (All / Local only / Known only / Core ports only / None) |
| **WiFi**   | подключение к сети (заодно время по NTP); включённый WiFi выключает Bluetooth |
| **MQTT**   | мост меша в интернет: публичный брокер по умолчанию или свой                |
| **Device** | **UTC** — выбор сдвига по городам, определяет все временные метки · **Clock** — часы устройства, `not set` до синхронизации; наберите `HH:MM`, чтобы выставить вручную, когда рядом нет телефона, NTP или ноды с GPS (часы сбрасываются при каждой перезагрузке, а сообщения, принятые без часов, остаются без времени) · **Screen** — автоотключение экрана (15 с … 5 мин или never; по умолчанию 5 мин) · **Font** — откуда взят Unicode-шрифт (`flash` / `sd` / `off`), только чтение |
| **Radio**  | **Onboard (Cap LoRa)** или **Companion via BLE** — см. ниже                 |

**↑/↓** — навигация, **Enter** — правка (или переключение), **ESC** — назад. Всё, что
касается радио (вся секция LoRa, WiFi / MQTT, режим Radio), применяется через перезагрузку.

В **режиме компаньона** секции Node и LoRa показывают — и правят — **подключённую ноду**
(см. ниже); WiFi / MQTT / Device / Radio остаются локальными для Cardputer.

### Режим компаньона (Settings → Radio → Companion via BLE)

Cardputer становится клавиатурно-экранным терминалом для **другой, стоковой Meshtastic-ноды**
(Heltec, T-Beam, RAK…) по Bluetooth — собственный LoRa-кап не нужен. Нода остаётся на своей
обычной прошивке; Cardputer говорит с тем же BLE-клиентским API, что и телефонное приложение.

1. **Скан** — после перезагрузки вы попадаете в *Find node*: ближайшие Meshtastic-ноды с
   именами и сигналом. `↑/↓` — выбор, **Enter** — подключение, **R** — повторный скан.
2. **Пейринг** — первое подключение спросит **PIN с экрана ноды** (цифры, Enter). Бонд
   запоминается; дальше подключение автоматическое, в том числе после перезагрузок.
3. **Подключено** — контакты и каналы ноды скачиваются, и вы в Чатах. Всё описанное выше —
   PKI-личные, галки доставки, реакции, ответы, каналы — теперь идёт через радио
   подключённой ноды.

Пока линк жив, **Bluetooth-руна** в шапке Чатов следит за подключением, а
**Settings → Radio** превращается в страницу статуса: состояние линка, принятые пакеты, id
ноды, сигнал линка (дБ), **заряд ноды** и сколько узлов синхронизировано. **R** —
принудительное переподключение, **F** — забыть ноду (назад в скан), **ESC** — просто выйти,
линк останется. Обрывы переподключаются в фоне сами.

**Настройки управляют нодой удалённо**, как телефонное приложение: **Name / Short** и имя
**Channel** применяются мгновенно по линку (объект канала ездит туда-обратно вместе с
ключом, так что переименование не сломает шифрование), а **Region / Preset / Frequency**
заставляют ноду сохраниться и перезагрузиться — экран уйдёт на страницу статуса Radio, и
линк вернётся сам. Безэкранные ноды обычно спариваются со стоковым PIN `123456`
(PIN-экран подскажет).

> 📱 У ноды один Bluetooth-слот — закройте телефонное приложение Meshtastic, пока
> Cardputer подключён, иначе они будут отбирать соединение друг у друга.

## Разное

- **История переживает перезагрузки** — кольцо сообщений (с реакциями и связями ответов)
  сохраняется во флеш, как и все настройки.
- **Звук + свет:** один бип + зелёная вспышка LED от избранных, синяя — от остальных; и
  никогда — жужжание на каждый пакет.
- **Автоотключение экрана** обесточивает рельсу дисплея целиком после заданного простоя.
  Любая клавиша будит (эта клавиша проглатывается); входящие экран не зажигают — уведомляют
  бип и LED, а бейджи непрочитанного ждут пробуждения.
- **Полный Юникод:** сообщения и имена на любом скрипте базовой плоскости (CJK, греческий,
  иврит, арабский, …) **плюс эмодзи-блоки** (U+1F000–U+1FBFF — реакции и сообщения от
  телефонных пользователей рисуются глифами Unifont вместо квадратиков; невидимые
  модификаторы — variation selector, ZWJ, тон кожи — пропускаются) через GNU Unifont —
  инсталлятор прошивает его в отдельную партицию, от пользователя ничего не нужно;
  esptool-пользователи пишут `unifont.bin` по `0x340000` или кладут его в корень SD-карты.
  Латиница и кириллица в любом случае остаются на быстром встроенном шрифте.
- **Телефонное приложение:** штатный Bluetooth API не тронут, официальное приложение
  Meshtastic спаривается с Cardputer как с любой нодой — всплывает экран с пин-кодом
  (разбудив дисплей), который надо ввести на телефоне. WiFi выключает Bluetooth, как в стоке.
- Шапка считает **узлы, слышанные за последние 24 часа** — флеш-леджер всех когда-либо
  встреченных идентичностей, так что число отражает живой меш и не упирается в лимиты
  хранения (пока часы не синхронизированы — фолбэк на счёт хранимых идентичностей). Сам
  список показывает 200 самых актуальных узлов с полными данными; в режиме компаньона —
  зеркало из 64 слышанных последними, а шапка считает всю базу подключённой ноды из
  синк-потока.
- Заряд чужих узлов компактная база этой сборки не хранит — только наша батарея
  (шапка/подвал) и, в режиме компаньона, батарея подключённой ноды.
