# Meshtastic ADV

[English](README.md) | **Русский**

**Клавиатурный клиент [Meshtastic](https://meshtastic.org) для [M5Stack Cardputer ADV](https://shop.m5stack.com/products/m5stack-cardputer-adv) — со своим Cap LoRa-1262 (SX1262) или через любую стоковую Meshtastic-ноду по Bluetooth.**

UI, написанный с нуля ради одной цели: чтобы **переписываться по мешу** с карманного QWERTY-устройства было по-настоящему удобно — без телефона — при этом внизу остаётся проверенный радио-стек Meshtastic. Нет LoRa-капа? [Режим компаньона](#режим-компаньона-чужая-нода-по-ble) превращает Cardputer в терминал для вашего Heltec / T-Beam / RAK.

## Зачем

У Cardputer есть настоящая клавиатура, цветной экран, динамик и SD-карта — но штатный экранный UI Meshtastic почти ничем из этого не пользуется, а набирать на нём сообщение мучительно. Этот проект **оставляет меш-движок Meshtastic нетронутым на 100%** (LoRa PHY, маршрутизация, PKI-криптография, база узлов, protobuf) и заменяет **только слой UI/ввода** аккуратным чатом, построенным вокруг клавиатуры.

Получается устройство, которое загружается сразу в рабочий мессенджер: выбрал контакт, набрал, нажал Enter.

## Что умеет

- **🗨️ Домашний экран с чатами** — загрузка сразу в переписки (личные + каналы), свежие сверху, у каждой превью последнего сообщения, время и бейдж непрочитанного. Открытие прыгает к первому непрочитанному.
- **💬 Личные сообщения** — открыл узел, набрал, отправил. PKI-шифрование как в телефонном приложении.
- **📢 Каналы** — чтение и отправка в любой канал, вперемешку с личными чатами.
- **✅ Статус доставки** — у каждого отправленного: *отправляется* (точка) → *доставлено* (зелёная галка по routing-ACK) → *ошибка* (красный ✗ с причиной). Широковещательные получают галку «отправлено».
- **⌨️ Кириллица + 😀 эмодзи** — русский набор на латинской клавиатуре через живой слой транслитерации (**Fn+L**); приём и отрисовка нелатинского текста и встроенных эмодзи; палитра по **Tab**.
- **🈶 Полный Юникод** — CJK, греческий, иврит, арабский… любой скрипт базовой многоязычной плоскости отрисовывается в сообщениях и именах через партицию с GNU Unifont, которую инсталлятор прошивает автоматически (латиница/кириллица остаются на быстром встроенном шрифте).
- **📇 Список узлов** — **Tab** показывает весь меш: полоски сигнала (по SNR), хопы, давность и роль, в фиксированных колонках.
- **🔎 Поиск контактов** — начните печатать, чтобы найти любой узел и начать новый чат.
- **⭐ Избранное** — пометьте контакты и каналы; они получают приоритетные уведомления.
- **🔔 Звук + свет** — один бип **и** зелёная вспышка LED от избранных; синяя — от остальных (никакого жужжания на каждый пакет).
- **🕘 Время** — компактное локальное `HH:MM` у каждого сообщения, сдвиг UTC выбирается по городам.
- **📡 WiFi + MQTT прямо с устройства** — подключение к WiFi и мост меша в интернет через MQTT (публичный брокер по умолчанию или свой), настраивается на самом устройстве — телефон не нужен. WiFi заодно выставляет часы по NTP.
- **💾 История переживает перезагрузки** — кольцо переписки сохраняется во флеш.
- **⚙️ Настройки на устройстве** — имя, регион, пресет модема, частота, канал, роль, хоп-лимит, мощность, режим ретрансляции, UTC, WiFi и MQTT — всё правится на устройстве (долгое нажатие **ESC**).
- **🔋 Автоотключение экрана** — дисплей гаснет после 15 с…5 мин простоя (на выбор); любая клавиша будит, уведомления при тёмном экране продолжают бипать и мигать LED.
- **📱 Телефонное приложение работает** — штатный Bluetooth API включён: спарьте официальное приложение Meshtastic (Cardputer покажет пин-код) и пользуйтесь им параллельно с экранным UI.
- **🔗 Режим компаньона** — нет LoRa-капа? Спарьте Cardputer с **любой стоковой Meshtastic-нодой** (Heltec, T-Beam, RAK…) по Bluetooth и используйте её радио: весь чат-UI выше — через чужую ноду. [Подробнее ниже.](#режим-компаньона-чужая-нода-по-ble)

## Установка

Проще всего — **[веб-инсталлятор](https://anton-vinogradov.github.io/meshtastic-adv/)** (ESP Web Tools):

1. Сначала прикрутите **антенну LoRa** к капу — *никогда не включайте усилитель без неё.*
2. Откройте страницу в десктопном **Chrome** или **Edge**, подключите Cardputer **data**-кабелем USB-C.
3. Нажмите **Install**. Если устройства нет в списке — зажмите **G0/BOOT** при подключении, потом отпустите.
4. После прошивки: долгое **ESC** → выставьте **Region** и **UTC**.

Предпочитаете CLI? Возьмите `firmware.factory.bin` из [последнего релиза](https://github.com/anton-vinogradov/meshtastic-adv/releases) и прошейте esptool'ом по смещению `0x0`; добавьте `unifont.bin` по `0x340000` для полного Юникода (или просто положите его в корень SD-карты).

## Управление

Всё с клавиатуры. Внизу каждого экрана — живые подсказки.

| Где | Клавиша | Действие |
|---|---|---|
| Чаты (дом) | **↑ / ↓** · **Enter** | курсор · открыть переписку |
| Чаты (дом) | **← / →** | в избранное / из избранного |
| Чаты (дом) | *печать* | поиск по всем узлам, новый чат |
| Чаты (дом) | **Tab** | полный список узлов (и обратно) |
| Переписка | *печать* · **Enter** | набрать ответ · отправить |
| Переписка | **Fn+L** | кириллица (транслит) вкл/выкл |
| Переписка | **Tab** | палитра эмодзи |
| Переписка | **↑ / ↓** | прокрутка истории |
| Переписка | **ESC** | назад |
| Везде | **долгое ESC** | настройки |

В настройках **↑/↓** — навигация, **Enter** — правка (для вкл/выкл — переключение), **ESC** — назад. Изменение Region/Preset/Frequency/Channel и WiFi/MQTT применяется через перезагрузку. Включённый WiFi выключает Bluetooth (поведение Meshtastic).

## Режим компаньона: чужая нода по BLE

Cardputer может работать **клавиатурно-экранным терминалом для отдельной Meshtastic-ноды** — вместо (или без) собственного LoRa-капа. Он подключается BLE-клиентом к штатному Bluetooth API ноды — тому же, которым пользуется телефонное приложение, — так что **нода остаётся на стоковой прошивке, ноль изменений**.

1. Долгое **ESC** → **Settings** → **Radio** → **Companion via BLE** → устройство перезагрузится в скан.
2. Выберите ноду из списка, введите **PIN** с её экрана.
3. Готово — контакты и каналы ноды скачиваются, и вы попадаете в чаты. Всё работает как обычно (PKI-личные, галки доставки, реакции, ответы), только через радио другой ноды.

Bluetooth-руна в шапке чатов показывает состояние линка; после обрывов он переподключается сам. **Settings → Radio** — заодно страница статуса (сигнал линка, заряд ноды, трафик; **R** — переподключить, **F** — забыть ноду). Назад — **Radio → Onboard (Cap LoRa)**.

Подключённую ноду можно и **настраивать с Cardputer**, как из телефонного приложения: Settings показывают значения *ноды*, **Name / Short / Channel** применяются мгновенно по линку, а **Region / Preset / Frequency** заставляют ноду сохраниться и перезагрузиться — линк восстановится сам.

> 📱 У ноды один Bluetooth-слот — закройте телефонное приложение Meshtastic, пока Cardputer подключён, иначе они будут отбирать его друг у друга.

## Архитектура

UI и меш-движок общаются по **тому же protobuf Client API** (`ToRadio` / `FromRadio`), что и телефонное приложение — но внутри процесса, на том же ESP32-S3, через собственную реализацию `PhoneAPI`.

```
  ┌─────────────────────────────┐        ToRadio / FromRadio        ┌──────────────────────────┐
  │   UI (этот проект, с нуля)  │  ───────────────────────────────▶ │   меш-движок Meshtastic  │
  │  клавиатура · экран · звук  │  ◀─────────────  protobuf  ─────── │  (upstream, без правок)  │
  └─────────────────────────────┘                                   │ LoRa · роутинг · крипто │
                                                                    └──────────────────────────┘
```

- **Функциональность как у стока** — всё, что умеет движок, доступно через тот же API, что у телефона.
- **UI полностью наш** — штатные `Screen` / canned-messages / модули ввода отключены.
- **Не форк** — upstream `meshtastic/firmware` лежит нетронутым git-сабмодулем; наш код живёт в `overlay/`, который копируется при сборке с двумя крошечными инъекциями в `main.cpp`. Обновление upstream не ломает UI.

Детали интерфейса — в [docs/interface.ru.md](docs/interface.ru.md), распиновка Cap LoRa-1262 — в [HARDWARE.md](HARDWARE.md) (англ.).

## Железо

| Часть | Детали |
|---|---|
| MCU | M5Stack Cardputer ADV — Stamp-S3A (ESP32-S3FN8, 8 МБ flash, **без PSRAM**) |
| Радио | Cap LoRa-1262 — Semtech **SX1262**, 868–923 МГц, внешняя антенна — *или* любая стоковая Meshtastic-нода по BLE (режим компаньона) |
| Экран | 1.14" 240×135 IPS |
| Ввод | QWERTY, 56 клавиш |
| Аудио | Кодек ES8311 + динамик |
| Прочее | microSD, RGB-светодиод, RTC, батарея |

> ⚠️ Никогда не включайте LoRa-кап без прикрученной антенны — усилитель может выйти из строя безвозвратно.

## Сборка из исходников

```sh
git clone --recursive https://github.com/anton-vinogradov/meshtastic-adv
cd meshtastic-adv
scripts/flash.sh        # синк оверлея, сборка, заливка, ресет
```

Нужен PlatformIO. Окружение сборки — `m5stack-cardputer-adv-advui`. `scripts/sync-overlay.sh` копирует `overlay/` в сабмодуль и применяет инъекции; `scripts/flash.sh` заодно разруливает ресет нативного USB.
